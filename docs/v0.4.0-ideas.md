# v0.4.0 Ideas

## Purpose

Capture compact next-milestone ideas based on dogfood findings after the `v0.3.0` release.

## Candidate Themes

- validation ergonomics
- better installer output
- memory conventions for very small repos
- richer examples for first-time adoption

## Recommended Direction

Prefer a small ergonomics-focused release that improves clarity around install-to-profile flow without adding runtime systems or semantic validation.

## Non-Goals

- runtime adapters
- semantic application validation
- deep repository scanning
- server, database, or background systems

## Deferred Ideas

- optional context warnings for missing non-required artifacts
- grouped installer summaries by folder or asset type
- more opinionated examples for tiny toy repositories and narrow feature goals
