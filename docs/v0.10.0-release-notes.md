# v0.10.0 Release Notes

**Experimental** — stable per-runtime support is **not** claimed.

## Summary

v0.10.0 ships **ai-engineering-harness** on npm with an interactive **NPX CLI** as the primary install UX. Provider detection **recommends** choices; the wizard does not silently auto-install.

## Added

- npm package `ai-engineering-harness` with bin aliases `ai-engineering-harness` and `aih`
- `bin/aih.js` — Node CLI wizard (`install`, `status`, `doctor`, `update`, `uninstall`)
- Provider checkbox selection (TTY raw mode) with numbered fallback
- `--provider` primary flag; `--runtime` alias
- [npx-cli-ux.md](npx-cli-ux.md), [npm-publish.md](npm-publish.md)

## Primary UX

```bash
npx ai-engineering-harness install
npx ai-engineering-harness status
npx ai-engineering-harness doctor
```

## Shell fallback (unchanged)

- `aih.sh` — lifecycle backend (bundled in npm package; CLI delegates here)
- `install.sh` — compatibility wrapper
- `aih.ps1` — experimental Windows bootstrap

## Install model

- `.ai-harness/` capability cache
- `.harness/` project state
- Runtime-specific entrypoint
- Private project: `.git/info/exclude` block (not `.gitignore`)

## Known limitations

- Shell backend still requires `sh` (Git Bash / WSL on Windows)
- Native JS install backend planned
- Antigravity shown as planned/disabled in wizard
- Multi-provider install runs sequential `aih.sh` invocations
- Manual fallback cannot combine with runtime-native providers in one wizard session
