# v0.10.2 Release Notes

**Experimental** — project-scoped `/harness:*` command surface.

## Added

- `runtime-command-catalog.js` — generates `.ai-harness/runtime-commands/`, `activation.md`, `manifest.json`
- Provider command entrypoints:
  - Claude: `.claude/commands/harness/*.md`
  - Cursor: `.cursor/commands/harness-*.md` + command mapping rule (fallback)
  - Gemini / OpenCode / AGENTS.md: alias mappings
- `commands/harness-status.md`, `commands/harness-doctor.md`
- [runtime-command-surface.md](runtime-command-surface.md)
- Status/doctor checks for command catalog and activation references

## Behavior

- `/harness:plan` → `.ai-harness/runtime-commands/harness-plan.md` → `.ai-harness/commands/harness-plan.md`
- Detection does not replace explicit command routing
- Uninstall removes provider command files; keeps `.ai-harness/runtime-commands/` unless `--remove-cache` / `--all`
