# v0.10.6 — Provider-native plugin packaging

## Summary

Aligns with [obra/superpowers](https://github.com/obra/superpowers): native commands come from **provider plugin manifests** at the package root, not invented project `.cursor/commands/` paths.

## Packaging added

- `.cursor-plugin/plugin.json` — `skills`, `commands`, `hooks`
- `.claude-plugin/plugin.json`
- `.codex-plugin/plugin.json`
- `gemini-extension.json`
- `.opencode/INSTALL.md`

## Project install behavior

| Provider | Native path | Invocation |
|----------|-------------|------------|
| OpenCode | `.opencode/commands/harness-plan.md` | `/harness-plan` |
| Claude | `.claude/commands/harness-plan.md` | `/harness-plan` |
| Cursor | rules only (fallback) | `/add-plugin` when marketplace ready |
| Gemini | extension context | ask harness:plan |
