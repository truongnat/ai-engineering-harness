# v0.10.5 — Honest command surface

## Problem

v0.10.2–0.10.4 generated a local command catalog **and** provider markdown files, but docs/README implied `/harness:plan` worked as a native slash command everywhere. File mappings do not automatically register slash commands in Cursor, Gemini, OpenCode, or generic agents.

## Fix

- **Local catalog always:** `.ai-harness/runtime-commands/`, canonical names `harness:plan`, …
- **Provider matrix (historical for v0.10.5):** Claude `native-unverified`; Cursor/Gemini/OpenCode/Generic `fallback-only`
- **Cursor:** only `.cursor/rules/ai-engineering-harness-commands.mdc` (no `.cursor/commands/` for native claim)
- **Manifest:** `commandSurface` with per-provider `mode` and `useInstruction`
- **CLI / status / doctor:** honest install plan and `lib/command-surface-report.js` output

## Docs

- [runtime-command-surface.md](runtime-command-surface.md) — full matrix and evidence notes

## Upgrade

```bash
npx ai-engineering-harness@0.10.5 update
```

On fallback providers, ask: **Use harness:plan for this repository.**
