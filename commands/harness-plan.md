# harness-plan

## Purpose

Translate a clarified goal into an actionable implementation plan before any coding or content changes begin.

## When To Use

- After discussion is complete
- Before any code or artifact modification
- When a previous plan is missing, stale, or invalidated

## Required Reads

- active goal, context, and discussion artifacts
- workspace map and dependency notes
- relevant workflow and template documents

## Skills To Use

- `using-harness`
- `writing-plans`
- `mapping-codebase` when affected areas need more detail

## Step-By-Step Workflow

1. Restate the agreed goal and affected scope.
2. Identify files, systems, and artifacts that will change.
3. Break the work into ordered, small, testable steps.
4. Define verification for each meaningful step.
5. Record risks, assumptions, and fallback options.
6. Save the plan as the source of truth for execution.

## Output Artifacts

- implementation plan
- task breakdown
- verification strategy

## Completion Gate

The command is complete when there is a concrete plan with ordered steps, explicit scope, and defined verification, and no implementation has started without it.
