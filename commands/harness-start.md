# harness-start

## Purpose

Start a work session with explicit context loading, artifact discovery, and operating discipline.

## When To Use

- At the start of a new task
- When resuming a paused task
- When switching to a different repository area

## Required Reads

- `AGENTS.md`
- `README.md`
- active `GOAL`, `STATE`, `CONTEXT`, and `PLAN` artifacts if present

## Skills To Use

- `using-harness`
- `mapping-codebase`
- `discussing-goals` when intent is still unclear

## Step-By-Step Workflow

1. Read the mandatory repository rules and active artifacts.
2. Confirm the current objective, constraints, and expected outcome.
3. Check whether an existing plan already exists and is still valid.
4. Map the workspace enough to identify the affected area.
5. Summarize the current state, open questions, and next command to run.

## Output Artifacts

- session start summary
- refreshed state notes
- clarified next action

## Completion Gate

The command is complete when the session has a clear goal, the relevant artifacts are loaded, and the next step is explicit instead of inferred.
