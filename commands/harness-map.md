# harness-map

## Purpose

Map the codebase, the active artifacts, and the engineering constraints before deeper planning or execution.

## When To Use

- At the beginning of a session
- Before proposing a plan
- When the workspace is unfamiliar
- When state drift is suspected

## Required Reads

- `AGENTS.md`
- `README.md`
- active `templates/STATE.md` or repository state artifact if present
- active goal, context, and roadmap artifacts if present

## Skills To Use

- `using-harness`
- `mapping-codebase`
- `remembering` when prior decisions must be loaded

## Step-By-Step Workflow

1. Read the current artifact set before touching code.
2. Inventory the top-level structure, important files, and recent state markers.
3. Identify the active goal, current constraints, and known decisions.
4. Trace the code or document areas most likely to be affected.
5. Summarize the map in terms of components, dependencies, and risks.
6. Record unknowns that must be resolved before planning.

## Output Artifacts

- updated context notes
- workspace map summary
- dependency or boundary notes

## Completion Gate

The command is complete when the relevant workspace structure, active artifacts, affected areas, and material unknowns are understood well enough to support a plan without relying on guesswork.
