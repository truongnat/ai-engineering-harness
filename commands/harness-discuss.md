# harness-discuss

## Purpose

Clarify goals, constraints, tradeoffs, and success criteria before turning work into a plan.

## When To Use

- When the request is ambiguous
- When scope must be reduced or split
- When multiple approaches are plausible
- Before planning any non-trivial change

## Required Reads

- active goal and requirements artifacts
- current roadmap or state artifacts if present
- prior remembered decisions that shape the approach

## Skills To Use

- `using-harness`
- `discussing-goals`
- `remembering` when existing decisions constrain the solution

## Step-By-Step Workflow

1. Restate the current ask in precise engineering terms.
2. Identify missing constraints, dependencies, and success criteria.
3. Resolve ambiguity through focused discussion.
4. Compare feasible approaches and highlight tradeoffs.
5. Recommend the leanest approach that satisfies the goal.
6. Capture the agreed direction in a discussion artifact.

## Output Artifacts

- discussion notes
- clarified scope and success criteria
- recommended approach

## Completion Gate

The command is complete when the goal, boundaries, and preferred approach are explicit enough to plan without inventing requirements mid-execution.
