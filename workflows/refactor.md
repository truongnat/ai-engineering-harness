# Refactor Workflow

Use this workflow for structural improvements that should preserve behavior.

1. Start and map the current boundaries.
2. Discuss the reason for refactoring and the non-goals.
3. Write a plan that defines invariants and checks.
4. Execute in small, behavior-preserving steps.
5. Verify that existing behavior still holds.
6. Ship with notes on what improved and what intentionally did not change.
7. Remember new boundaries or conventions if they matter later.
