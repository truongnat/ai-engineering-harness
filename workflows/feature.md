# Feature Workflow

Use this workflow for new capabilities or meaningful behavior changes.

1. Start and map the affected area.
2. Discuss scope, constraints, and success criteria.
3. Write a plan that defines implementation and verification.
4. Execute in small steps and prefer test-first evidence when behavior changes.
5. Verify the feature against the success criteria.
6. Ship with notes on follow-ups.
7. Remember durable decisions and constraints.
